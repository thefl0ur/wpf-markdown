### Markdown renderer

Используемые пакеты
* [Markdig.Wpf](https://github.com/Kryptos-FR/markdig.wpf)

### Как работает

В проект устанавливается пакет **Markdig.Wpf**, на формре размещается компонент для отображения `markdig:MarkdownViewer`, 
в свойство `Markdown` которого передается текст, который будет отрендерен.

Компонент содержит команды Hyperlink и Image (нами не применяется, поэтому не изучался)

Т.к. Hyperlink это RoutedCommand, а MVVM использует RelayCommand, что бы связать их нужно отойти от MVVM-модели.

во ViewModel создается RelayCommand с обработчиком.
в code-behind view создается событие, внутри которого будет получаться ViewModel данного окна и вызываться команда.

### Особенности

На сервисе dist используется форматирование вида `## что-то ##` для выравнивания по центру. Не поддерживается

Что бы текст корректно отображался как жирный, между текстом и символами не должно быть пробелов:

    ** НЕ ПРАВИЛЬНО**
    ** НЕ ПРАВИЛЬНО **
	**ПРАВИЛЬНО**
	
Не работают следушие функции:
 
 * Переход по частям документа по ссылкам, сама ссылка обрабатывается как обычная ссылка
 * Не работают Emoji
 * типография типа `(c) (C)`  не рендерится
 * картинки не рендерятся (по ссылки,, локально не тестировалось)
